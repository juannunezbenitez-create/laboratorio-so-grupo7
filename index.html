Fases de Implementación Técnica

FASE 01

Componente 1: Virtualización con Linux

Zorin OS 17 (Gráfica) Ubuntu 24.04 (Consola)

Instalación y configuración de dos máquinas virtuales interconectadas por Red NAT. Se aplicó particionado manual y pruebas de conectividad remota mediante el protocolo SSH.

Evidencias Esenciales

Estructura de Particionado (lsblk)

Asignación de Interfaces de Red

Túnel de Conexión SSH Funcional

Comandos Principales

ip a                        # Ver interfaces de red e IPs asignadas
lsblk                       # Ver estructura y jerarquía de particiones
ssh usuario@ip_vm_consola   # Establecer sesión remota segura por SSH
FASE 02

Componente 2: Contenedores Docker

Base: debian:13 Docker Compose

Estructuración y despliegue automatizado de una arquitectura desacoplada. El frontend corre sobre un servidor Nginx en el puerto 80 y la API del backend se gestiona mediante Python en el puerto 5000.

Evidencias Esenciales

Contenedores Activos (docker ps)

Acceso Exitoso al Frontend

Petición HTTP a API Backend

Comandos Principales

docker compose up -d        # Instanciar y encender microservicios aislados
docker ps                   # Consultar estado y puertos de contenedores activos
curl http://localhost:5000  # Probar respuesta HTTP del microservicio Python
FASE 03

Componente 3: Orquestación con Kubernetes

Minikube NodePort 30080

Orquestación local del despliegue. Se crearon los manifiestos de configuración para la automatización y replicación elástica del servidor web, exponiendo el servicio al exterior de la MV por medio del puerto estático 30080.

Evidencias Esenciales

Mapeo del Servicio NodePort

Verificación Elástica de 3 Réplicas

Sitio Web Expuesto en Producción

Manifiestos y Comandos

• deployment.yaml — Configurado con 3 réplicas funcionales listas en el clúster.

• service.yaml — Expone NodePort amarrado estrictamente al puerto 30080.

minikube start                              # Iniciar entorno local clúster K8s
kubectl apply -f laboratorio_k8s/           # Aplicar manifiestos YAML de la carpeta
kubectl scale deployment nginx --replicas=3 # Escalado elástico horizontal de Pods

Conclusiones del Proyecto

1. Pasar todo a contenedores nos salvó la vida [Aprendizaje Principal]

El aprendizaje principal fue ver cómo cambia la cosa al separar el Frontend y el Backend en contenedores de Docker en lugar de meter todo en un solo bloque. Al montarlos sobre Debian 13 nos dimos cuenta de que cada servicio trabaja por su lado sin interferir con el otro, lo que hace que todo sea mucho más ordenado y fácil de manejar.

2. Los errores de red y de paquetes nos hicieron sufrir, pero se solucionaron [Dificultad]

La mayor dificultad fue cuando la máquina virtual se bloqueó y no quería instalar Git por culpa de unos paquetes rotos, y también cuando los Pods de Nginx en Minikube no querían conectar. Al final lo arreglamos metiendo el paquete a la fuerza con comandos manuales y amarrando bien los servicios al puerto 30080 con NodePort para que el tráfico saliera sin problemas.

3. Para la próxima, se necesita una mejor computadora [Recomendación]

Como recomendación para los que vayan a hacer este proyecto después, seria bueno tener una buena máquina con bastante memoria RAM. Correr VirtualBox, la máquina virtual, Docker y encima prender Minikube al mismo tiempo pone la computadora lentísima si no se le asignan buenos recursos desde el principio.
